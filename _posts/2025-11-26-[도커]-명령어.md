#### 1. 컨테이너 라이프사이클 (Lifecycle)


이미지로부터 컨테이너 객체를 생성, 실행, 정지, 삭제하는 핵심 제어 명령.


| **명령어**                        | **기술적 동작 설명**                                                |
| ------------------------------ | ------------------------------------------------------------ |
| **`docker create [옵션] [이미지]`** | Writable Layer 생성 및 설정 주입. 프로세스는 실행하지 않음. (`Created` 상태)     |
| **`docker start [컨테이너]`**      | `Created` 또는 `Exited` 상태인 컨테이너의 메인 프로세스(PID 1) 시작. (`Up` 상태) |
| **`docker run [옵션] [이미지]`**    | `create`와 `start`를 순차적으로 수행. 이미지가 없으면 `pull` 수행.             |
| **`docker stop [컨테이너]`**       | `SIGTERM` 시그널을 전송하여 프로세스 Graceful Shutdown 수행.               |
| **`docker rm [컨테이너]`**         | `Stopped` 상태인 컨테이너의 Writable Layer 및 메타데이터 영구 삭제.            |
| **`docker ps`**                | 현재 `Running` 상태인 컨테이너 프로세스 목록 출력.                            |
| **`docker ps -a`**             | `Exited` 상태를 포함한 모든 컨테이너 객체 목록 출력.                           |
| **`docker pause [컨테이너]`**      | 컨테이너 내 모든 프로세스를 일시 정지(SIGSTOP). (메모리 상주)                     |
| **`docker unpause [컨테이너]`**    | 일시 정지된 프로세스 재개.                                              |

#### 2. 이미지 빌드 및 관리 (Image Operations)


Dockerfile 기반 빌드 및 레지스트리, 아카이브 관리 명령.


| **명령어**                             | **기술적 동작 설명**                               |
| ----------------------------------- | ------------------------------------------- |
| **`docker build -t [태그] .`**        | 현재 Context의 `Dockerfile` 명세에 따라 이미지 레이어 빌드. |
| **`docker images`**                 | 로컬 스토리지에 저장된 이미지 및 태그 목록 조회.                |
| **`docker rmi [이미지]`**              | 로컬 이미지 레이어 삭제. (참조 중인 컨테이너가 없어야 함)          |
| **`docker tag [Src] [Target]`**     | 기존 이미지 ID에 새로운 참조(Alias) 태그 추가.             |
| **`docker push [이미지]`**             | 로컬 이미지를 원격 레지스트리로 업로드.                      |
| **`docker pull [이미지]`**             | 원격 레지스트리에서 이미지 레이어 다운로드.                    |
| **`docker save -o [파일.tar] [이미지]`** | 이미지를 레이어 정보 포함하여 tar 아카이브로 추출. (오프라인 이관용)   |
| **`docker load -i [파일.tar]`**       | tar 아카이브에서 이미지 레이어 및 메타데이터 복원.              |
| **`docker history [이미지]`**          | 이미지의 레이어별 생성 명령어, 크기, 생성 시각 조회.             |
| **`docker commit [컨테이너] [이미지]`**    | 실행 중인 컨테이너의 현재 상태를 스냅샷 떠서 새 이미지로 생성. (비권장)  |

#### 3. 관측 및 디버깅 (Observability & Debugging)


컨테이너 내부 상태 확인, 로그 분석, 프로세스 진입 명령.


| **명령어**                            | **기술적 동작 설명**                                         |
| ---------------------------------- | ----------------------------------------------------- |
| **`docker logs -f [컨테이너]`**        | 컨테이너의 STDOUT/STDERR 스트림을 실시간 출력.                      |
| **`docker exec -it [컨테이너] [Cmd]`** | 실행 중인 컨테이너 네임스페이스 내에서 별도 프로세스 실행.                     |
| **`docker inspect [컨테이너/이미지]`**    | 객체의 Low-level 설정(IP, Mount, Env 등)을 JSON 포맷으로 출력.     |
| **`docker stats`**                 | 모든 컨테이너의 리소스(CPU, Mem, Net I/O) 사용량 실시간 스트림 모니터링.     |
| **`docker top [컨테이너]`**            | 컨테이너 내부 프로세스 목록(PID)을 호스트 관점에서 출력.                    |
| **`docker diff [컨테이너]`**           | 이미지 생성 시점 대비 파일시스템 변경 내역(Added, Changed, Deleted) 출력. |
| **`docker cp [Src] [Dest]`**       | 호스트와 컨테이너 간 파일/폴더 양방향 복사.                             |

#### 4. 시스템 및 리소스 관리 (System Management)


디스크 공간 확보 및 Docker 데몬 관리 명령.


| **명령어**                      | **기술적 동작 설명**                                           |
| ---------------------------- | ------------------------------------------------------- |
| **`docker system prune -a`** | 사용되지 않는 모든 리소스(Dangling 이미지, 중지된 컨테이너, 네트워크, 캐시) 일괄 삭제. |
| **`docker container prune`** | `Stopped` 상태인 컨테이너 객체만 일괄 삭제.                           |
| **`docker image prune`**     | 이름 없는(Dangling) 이미지 레이어만 일괄 삭제.                         |

#### 5. 네트워크 관리 (Network)


컨테이너 간 통신을 위한 가상 네트워크 인터페이스 제어.


| **명령어**                                    | **기술적 동작 설명**                                |
| ------------------------------------------ | -------------------------------------------- |
| **`docker network create [이름]`**           | 사용자 정의 네트워크(Bridge/Overlay 등) 생성. DNS 해석 지원. |
| **`docker network ls`**                    | Docker 데몬에 생성된 네트워크 드라이버 목록 조회.              |
| **`docker network inspect [이름]`**          | 특정 네트워크의 서브넷, 게이트웨이, 연결된 컨테이너 정보 조회.         |
| **`docker network connect [망] [컨테이너]`**    | 실행 중인 컨테이너에 새로운 네트워크 인터페이스(NIC) 동적 추가.       |
| **`docker network disconnect [망] [컨테이너]`** | 컨테이너에서 특정 네트워크 인터페이스 제거.                     |

#### 6. 오케스트레이션 (Docker Compose)


다중 컨테이너 애플리케이션 정의 및 제어.


| **명령어**                      | **기술적 동작 설명**                        |
| ---------------------------- | ------------------------------------ |
| **`docker compose up -d`**   | 정의된 서비스, 네트워크, 볼륨 생성 및 백그라운드 실행.     |
| **`docker compose down`**    | `up`으로 생성된 컨테이너 정지/삭제 및 네트워크 리소스 제거. |
| **`docker compose ps`**      | Compose 프로젝트 내 서비스들의 상태 조회.          |
| **`docker compose logs -f`** | 프로젝트 내 모든 서비스의 로그 스트림 집계 출력.         |

#### 7. 컨텍스트 및 원격 제어 (Context)


로컬 CLI를 사용하여 원격 Docker 데몬 제어.


| **명령어**                               | **기술적 동작 설명**                    |
| ------------------------------------- | -------------------------------- |
| **`docker context create [이름] [옵션]`** | 원격 Docker 호스트(SSH/TCP) 연결 정보 등록. |
| **`docker context use [이름]`**         | CLI 타겟을 지정된 컨텍스트로 전환.            |
| **`docker context ls`**               | 등록된 컨텍스트 목록 확인.                  |
